# uni-app SVG 资产主权

本 skill 负责把图标、空状态图和轻量图形收归项目内，不把关键视觉资产交给外部平台。

## 适用范围

- 业务图标
- 空状态图形
- 组合图形或轻插画
- 图标替换
- SVG 组件封装

## 默认策略

1. 默认产出 `SVG`，不是 `PNG`。
2. 默认放到项目静态资源目录，例如 `src/static/icons/`。
3. 默认补薄组件或复用现有组件，不在页面里散落硬编码路径。
4. 默认至少接到一个真实页面验收。

## 资产主权规则

优先使用项目内实现的场景：

- 品牌型业务图标
- 高复用入口图标
- 关键空状态图
- 能明显影响产品气质的小资产

仅在以下情况允许优先复用外部方案：

- 系统级通用图标，`uni-icons` 已足够
- 品牌无要求且生命周期很短
- 项目明确要求兼容现有 iconfont 体系

## 目录与命名

- 文件名统一使用短横线：`ic-{name}.svg`
- 目录建议：
  - `src/static/icons/common/`
  - `src/static/icons/{domain}/`

## 工作流程

1. 先判断是否必须新增 SVG，而不是先画。
2. 检查项目是否已有图标组件、目录和命名规则。
3. 需要新增时，优先成批补齐同风格资产，不单点插入异类图标。
4. 同批超过 3 个图标时，优先补预览区或集中使用点。
5. 接入后检查真实页面，而不是只检查资源文件。

## 画法约束

- 默认统一 `viewBox`，例如 `96x96`
- 同批资产描边粗细、留白比例、圆角倾向一致
- 缩到 `18-28px` 仍要清晰
- 尽量减少碎细装饰
- 默认不引入多色插画风格；除非明确是空状态或场景图

## 接入约束

- 页面不要大量直写资源路径，优先走组件映射。
- 图标组件只负责 `name`、`src`、尺寸和轻量展示。
- 如果替换已有入口图标，尽量同步同类入口，避免半套风格。

## 验收

- 小尺寸是否清晰
- 深浅主题下是否可见
- 真实页面节奏是否协调
- 至少运行一次 `npm run type-check`

## 与其他 skill 协作

- 接入页面、组件、样式时，同时使用 `uni-component-style-flow`
- 有品牌规范时，叠加品牌 UI skill
- 在 `uni-dev-flow` 下，SVG 相关需求默认触发本 skill
