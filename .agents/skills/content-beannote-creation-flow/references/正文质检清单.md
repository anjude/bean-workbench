# 正文质检清单

> 只放交付前检查项，不放写作准则，不放模板。

## 文件检查

- [ ] 主题目录使用 `YYYY-MM-DD_主题简述`
- [ ] 公众号文章.md 开头包含选题区（写给谁、一句话结论、核心问题、标题草案）
- [ ] 公众号文章：有 `公众号文章.md`（含选题区+正文）、`发布检查清单.md`、`复盘.md`
- [ ] 小红书图文：有 `图文脚本.md`（含选题区+正文）、`纯享正文.md`、`发布检查清单.md`、`复盘.md`
- [ ] 额外脚本、跨平台正文只在需要时生成

## 硬禁令（不通过就重写）

- [ ] 正文无第二人称"你"（互动问题可例外；选题区不计入）
- [ ] 无中文破折号 `——`
- [ ] 无英文 em dash `—`
- [ ] 无角引号 `「」` `『』`
- [ ] 无装饰项目符号 `•` `·`
- [ ] 无用引号包裹关键词或术语
- [ ] 无一句话里自相矛盾（不左右互搏）
- [ ] 无无意义平衡句（"当然是好事"、"这本身没问题"、"并不是坏事"）
- [ ] 无和稀泥句式（"谁对？都对"、"各有道理"、"只是场景不同"）
- [ ] 无空洞感慨（"这个坑踩过的人不少"、"很多人吃过亏"类无信息量垫话）
- [ ] 无凭空否定：对文中没出现的负面判断做否定（如"不是补窟窿"），属预设对立
- [ ] 无保本保收益承诺、"稳赚""无风险"等绝对表述
- [ ] 无直接喊买/卖具体标的
- [ ] 无开户或交易诱导
- [ ] 已保留免责声明

## 爆款检查（每项都要有至少一项达标）

- [ ] 标题有反常识感或强情绪，让人"愣一下"
- [ ] 开头前50字有冲突、情绪或硬信息，不慢热
- [ ] 有至少一句让人想截图/转发的金句（精彩引言）
- [ ] 全文有主判断，不是资料摘要或温吞科普
- [ ] 有反差或冲突：常识vs现实、情绪vs数据、热闹vs风险

## 信息量检查

- [ ] 每段都有新增信息量（新事实/新观察/新判断/新例子/新情绪）
- [ ] 强表达后面有事实、例子、数据、方法或边界支撑，不是空喊口号
- [ ] 至少有一个具体支撑物：数据、案例、场景、类比、框架、步骤、边界
- [ ] 读者看完能带走一个判断方式，而不是只记住一个结论

## 文风检查（去AI味，不去个性化）

- [ ] 没有解释笑点或解释情绪
- [ ] 没有"表面看...往深一点看..."等升华模板
- [ ] 没有三段以上同动词排比（"变了X，变了Y，变了Z"）
- [ ] 没有升华式段尾（"X是Y的轮廓/缩影/注脚"）
- [ ] 没有主持人预告句（"X是一个分界线/转折点/里程碑"）
- [ ] 结尾无空总结（整节只是在换句话复述前文→整节删掉）
- [ ] 粗体 3-5 处（小节标题加粗不计入），不泛滥
- [ ] 自述占比不过高（"我"字全文不超过3次，人设开场白除外）

## 自进化检查

- [ ] 用户反馈已先落实到当前稿
- [ ] 已判断本次问题是一次性修改还是规则盲区
- [ ] 如属规则盲区，已同步更新 memory、skill 或 references 中对应文件
- [ ] 当前主题 `复盘.md` 已记录用户反馈和规则沉淀

## 发布前确认

- [ ] 标题最终确定
- [ ] 事实源日期和说法已核对
- [ ] 如使用媒体截图或引用，来源清楚
- [ ] 留言引导不使用"你是不是也..."这类AI句式
