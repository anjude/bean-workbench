# 碳基空间品牌 UI 规范

本 skill 不只是配色和样式参考，而是 carbon 的品牌治理层。

## 一句话

关了灯的两个人的空间，有微光，有呼吸，但很安静。

## 关键词

- 夜空
- 萤火
- 细胞
- 星图
- 磨砂

## 品牌目标

carbon 的视觉要同时传达：

1. 私域感：这是两个人的地方，不是公共广场。
2. 温度感：有机、温暖、生物性，不是冰冷工具。
3. 安静感：表达克制，不吵不闹。

## 结构治理

看到 carbon 页面时，先检查：

- 页面是否有单一视觉重心
- CTA 是否足够克制
- 信息是否围绕“关系”和“空间”组织
- 空状态是否像在邀请进入，而不是在报错
- 页面是否像一个稳定空间，而不是功能清单

如果答案是否定的，先调结构，不先调颜色。

## 审美边界

允许：

- 深色或低照度背景
- 微光强调
- 大圆角
- 轻磨砂
- 缓慢呼吸型动效
- 留白驱动层次

不允许：

- 企业后台式线框堆叠
- 游戏化卡通符号泛滥
- 过多彩色标签和强对比按钮
- 为了“高级感”堆满透明度、阴影和模糊
- 同屏出现多个竞争视觉中心

## 图标与资产语义

- 图标应像“信号”或“坐标”，不是宣传插画
- 空状态图应像“邀请进入”，不是“系统没数据”
- 分享图、背景纹理、轻插画都应服务安静和私密，不服务热闹
- 业务图标默认走 `uni-svg-flow + carbon-icon-flow`

## 动效约束

- 核心词：呼吸，不是弹跳
- 只允许少量循环动效
- 优先 `opacity`、`transform`
- 交互反馈尽量小而明确
- 不为了情绪氛围打断主流程

## 页面决策规则

当多个方案都可行时，优先级如下：

1. 更像一个空间
2. 更安静
3. 更容易理解
4. 更少依赖外部资产
5. 更少打断用户

## 与其他 skill 协作

- 总控流程由 `uni-dev-flow` 调度
- 页面与样式执行交给 `uni-component-style-flow`
- 图标和小资产交给 `uni-svg-flow` 与 `carbon-icon-flow`
