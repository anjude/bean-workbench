# Skill：知识库整理 — 核心思想与认知框架

## 用途

将各类来源（书籍、演讲、访谈、文章、播客等）中的心理偏误、认知框架、投资洞见，整理为结构化的知识库条目，存入 `business-repo/beannote/04_knowledge_base/`，供内容创作直接调用。

**触发场景示例：**
- "整理《穷查理宝典》人类误判心理学"
- "把《思考快与慢》的核心框架整理进知识库"
- "巴菲特那篇股东信里的投资原则整理一下"
- "把这个访谈里的核心观点提炼出来"
- "把 Naval 的推文合集整理成知识库"

---

## 执行流程

### Step 1：确认来源信息

根据用户输入，明确以下信息（不确定时主动询问）：

| 字段 | 说明 |
|------|------|
| 来源类型 | 书籍 / 演讲 / 访谈 / 文章 / 播客 / 推文合集 / 其他 |
| 来源名称 | 书名、演讲标题、访谈名称等 |
| 作者/主讲人 | 用于文件命名和归因 |
| 具体章节/段落 | 如"第十一讲"、"第三章"、"某期节目"（无则略） |
| 条目数量 | 有序列表型（如25条）/ 非序列型（自由提炼） |
| 知识库路径 | 默认 `{workspace}/business-repo/beannote/04_knowledge_base/` |

### Step 2：判断内容类型，选择整理模式

根据来源内容，选择对应整理模式：

#### 模式 A：序列型条目（有明确编号的清单）
适用：芒格人类误判、卡尼曼偏误列表等有序列的内容。
→ 按序号逐条整理，每条独立成段。

#### 模式 B：主题型框架（无序列，按主题提炼）
适用：巴菲特股东信、Naval 推文、TED 演讲等无固定序号的内容。
→ 自主归纳为 3-8 个核心主题，每个主题下展开说明。

#### 模式 C：混合型（既有核心清单，也有延伸论述）
适用：书籍章节、长篇访谈等。
→ 先列核心条目清单，再在每条下展开。

### Step 3：整理每条/每主题的标准结构

**模式 A 的每条结构：**
```
### N. 偏误/洞见名称
**核心**：一句话说清本质机制或核心观点。
**投资/理财场景**：至少一个具体场景，有行为 + 有结果，散户能对号入座。
```

**模式 B/C 的每个主题结构：**
```
### 主题名称
**核心观点**：2-3句话说清这个主题的核心论断。
**实际含义**：用普通人的语言解释，避免学术腔。
**投资/理财应用**：具体场景举例，要有画面感。
```

**通用写作要求（所有模式）：**
- 不要照抄原文定义，必须翻译成投资/理财语境
- 举例要具体，有行为有结果，不要笼统
- 名称保留原文术语，可在括号内加俗称
- 用口语化表达，删除书面腔

### Step 4：完整文件结构

```markdown
# {来源名称} — {核心主题} 整理

> 来源：{来源类型}《来源名称》{章节/时间戳（如有）}
> 作者/主讲人：{姓名}
> 整理日期：YYYY-MM-DD
> 用途：内容创作素材 / 投资行为分析 / 爆款选题挖掘

---

## 核心论点

（简述作者/主讲人的整体主张，2-4句话，不超过100字）

---

## {N条偏误 / 核心框架 / 主要洞见}

（按模式 A/B/C 展开，格式见 Step 3）

---

## 投资应用总结

| 类别/主题 | 典型投资错误 | 应对方法 |
|----------|-------------|---------|
| ...      | ...         | ...     |

---

## 内容创作方向

- **爆款选题**：列出最有潜力的3-5条，说明理由（用停/点/传三维度评估）
- **创作角度**：如何将抽象概念转化为散户能感同身受的场景
- **系列化思路**：是否适合做多篇系列，建议切入角度

---

*参考来源：{来源类型}《来源名称》{章节（如有）}*
```

### Step 5：文件命名规范

格式：`{作者姓}_{核心主题}_{条数或形态}.md`

示例：
- `芒格_人类误判心理学_25条.md`
- `巴菲特_1995股东信_投资原则.md`
- `Naval_财富创造洞见.md`
- `卡尼曼_认知偏误核心框架.md`

存放路径：`{workspace}/business-repo/beannote/04_knowledge_base/`

### Step 6：爆款潜力评估（整理完成后必做）

整理完毕后主动输出爆款潜力简评表，帮用户快速确定下一步选题：

| 条目/主题 | 爆款潜力 ⭐1-3 | 理由 |
|----------|--------------|------|
| ...      | ...          | ...  |

评判维度：
- **停**：是否有反常识感，让用户"愣一下"
- **点**：是否精准戳到散户/目标用户的痛点
- **传**：是否有争议性/让人想评论或转发

---

## 质量自检清单

完成整理后，逐项检查：

- [ ] 每条/每主题都有具体的投资/理财场景举例（非泛泛而谈）
- [ ] 名称/标题保留原文术语（可加俗称）
- [ ] 用口语化表达，删除书面腔和学术腔
- [ ] 文件末尾有「投资应用总结表」
- [ ] 文件末尾有「内容创作方向」及爆款选题建议
- [ ] 文件已存入 `business-repo/beannote/04_knowledge_base/` 目录
- [ ] 文件命名符合规范（作者_主题_形态.md）
