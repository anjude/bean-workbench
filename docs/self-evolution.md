# 自进化机制

## 目标

让工作台能根据真实使用过程持续改进，而不是只依赖一次性规则设计。

## 触发条件

满足任一情况时，应考虑触发自进化：

1. Agent 连续两轮以上没有识别清楚用户意图。
2. 某类操作路径的错误、返工或验证失败经常复现。
3. 业务需求中出现可以复用到未来需求的流程、约束或知识。
4. 热任务频繁出现，但缺少清晰路由、上下文或验收方式。
5. 用户明确指出工作台规则、skill、知识库或脚本需要优化。

## 处理流程

1. 先把问题记录到 [痛点收集](pain-points.md)。
2. 如果只是临时上下文，更新 [任务登记](../task/registry.md)。
3. 如果是稳定知识，沉淀到 `knowledge-base/`。
4. 如果是稳定流程，沉淀为 `.agents/skills/`。
5. 如果是重复命令或机械操作，沉淀为 `script/` 或 `Makefile` target。

## 升级判断

- 一次性问题：只记录在任务文档。
- 两次以上复现：进入痛点收集。
- 三次以上复现且路径稳定：升级为知识库、skill 或脚本。

## 复盘要求

完成业务需求后，Agent 应主动判断是否需要更新工作台。如果不需要，应在最终回复中保持简洁，不额外制造文档。
