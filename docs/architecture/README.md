# 架构规范

## 目标

定义工作台目录职责和演进边界，避免工作台、知识库、任务记录和业务仓职责混淆。

## 分层

- 根目录：只放入口文件和通用工具配置。
- `docs/`：工作流、模板、架构规范和目标管理。
- `.agents/skills/`：可触发的 Agent skill。
- `knowledge-base/`：稳定知识沉淀。
- `task/`：任务上下文、热任务和复盘。
- `archive/`：历史任务、过期规则和阶段性材料归档。
- `script/`：可执行辅助脚本。
- `business-repo/`：业务仓集合。

## 演进原则

1. 新能力先进入任务记录或知识库。
2. 复现稳定后再升级为 skill。
3. 机械重复操作再升级为脚本。
4. 根目录保持克制，新增长文档优先放入 `docs/`。
5. 已完成且不再活跃的任务、规则草稿或阶段材料应迁入 `archive/`。
