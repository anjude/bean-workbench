# 知识库规范

## 目标

`knowledge-base/` 用于沉淀跨需求可复用的业务知识、系统知识和协作经验，服务于后续业务仓代码和文档产出。

## 分层方式

- `knowledge-base/common/`：公共业务知识、通用产品理解、跨仓可复用约束。
- `knowledge-base/business-repo/`：按业务仓沉淀的知识索引和系统理解。
- `knowledge-base/systems/`：跨仓系统、基础设施、工具链和平台级知识。

## 写入原则

1. 只写稳定、可复用、能减少未来理解成本的内容。
2. 不复制业务仓源码或大段接口文档，只记录索引、结论、约束和入口。
3. 每条知识应尽量说明来源、适用范围和更新时机。
4. 不确定的信息要标注为待验证。

## 自进化关系

当同类问题复现或需求结束后出现稳定经验，应先进入知识库；如果知识进一步变成固定执行流程，再升级为 skill 或脚本。
